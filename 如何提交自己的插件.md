# 如何提交自己的 DataEase 插件

欢迎为 DataEase 生态贡献新的插件。本文档将结合当前插件仓库的实际结构，说明如何开发、整理并提交一个可用的 DataEase 插件。

## 📋 提交前准备

### 插件类型要求

当前 DataEase 插件主要分为以下三类：

- **数据源插件**：用于接入新的数据库或数仓
- **数据填报插件**：用于扩展数据填报能力
- **图表插件**：用于扩展图表、可视化或组件能力

### 插件基本要求

- 必须具备明确的功能定位，能够解决 DataEase 中的实际使用场景
- 不得包含恶意代码、后门逻辑或不必要的敏感信息

## 🏗️ 插件目录结构规范

每个插件需要按照以下目录结构组织：
```
/
├── data.yaml             # 插件配置文件
├── logo.png              # 插件图标 (建议 50x50px)
├── README.md             # 插件说明文档
└── 1.0.0/                # 版本目录 (此目录只能包括一个.jar文件)
    └── 插件文件.jar       # 插件文件
```

## 📝 配置文件说明

### data.yaml 配置文件

这是插件的核心配置文件，包含插件的基本信息：

```yaml
name: TDEngine 数据源插件
tags:
  - 数据源插件
title: TDEngine 数据源插件
edition: enterprise
dataease_version: 2.10.21
description: 基于 TDEngine 3.3.6.13 开发
```

**字段说明：**
- `name`: 插件的显示名称
- `tags`: 插件分类标签，从预定义标签中选择
- `title`: 插件简短标题
- `edition`: 插件适用的 DataEase 版本类型
- `dataease_version`: 插件要求的最低 DataEase 版本
- `description`: 插件功能的详细描述
## 🎯 开发指南

### 数据源插件开发建议

如果你提交的是数据源插件，建议至少覆盖以下内容：

1. **数据源配置**
   - 支持基础连接参数配置
   - 明确 JDBC URL 格式
   - 明确驱动依赖和版本

2. **连接与查询能力**
   - 能正常建立连接
   - 支持基础查询、元数据读取、字段识别
   - 具备必要的异常处理

3. **插件元数据**
   - `config.type`、`config.name`、`config.category` 等字段完整
   - `driverPath` 指向正确
   - 如果有前端页面，`staticMap` 需与前端导出模块保持一致

### 数据填报插件开发建议

如果你提交的是数据填报插件，建议重点关注：

1. **表结构识别**
   - 能识别字段类型、主键、索引等基础信息

2. **DDL / DML 能力**
   - 支持新增、编辑、删除、建表等核心操作
   - SQL 方言与目标数据库保持一致

3. **兼容性**
   - 字段类型映射准确
   - 特殊字段、保留字、引号转义处理完善

### 图表插件开发建议

如果你提交的是图表插件，建议至少包括：

1. **图表元数据**
   - `config.chartValue`
   - `config.chartTitle`
   - `config.category`
   - `config.render`

2. **前端编辑器能力**
   - 数据配置面板
   - 样式配置面板
   - 高级配置面板

3. **静态资源映射**
   - `staticMap` 中的入口与编辑器模块可正确加载
   - 打包后产物能够被后端正确复制并发布

## 📦 提交流程

### 1. Fork 仓库

点击右上角的 "Fork" 按钮，将仓库复制到您的 GitHub 账号下。

### 2. 创建插件目录

在 `extensions/` 目录下创建您的插件目录：

```bash
mkdir extensions/your-extension-name
cd extensions/your-extension-name
```

### 3. 编写插件文件

按照上述结构创建必要的文件：
- 编写 `data.yaml` 配置文件
- 添加插件图标 `logo.png`
- 创建详细的 `README.md`
- 实现插件功能代码

### 4. 本地测试

在提交前请确保：
- 配置文件格式正确
- 插件功能正常运行
- 文档完整清晰
- 遵循编码规范

### 5. 提交 Pull Request

1. 提交您的修改：
```bash
git add .
git commit -m "添加 [插件名称] 插件"
git push origin main
```

2. 在 GitHub 上创建 Pull Request
3. 在 PR 描述中详细说明：
   - 插件的功能和用途
   - 测试情况
   - 相关依赖

## ❓ 常见问题

**Q: 插件审核需要多长时间？**
A: 通常 3-7 个工作日，复杂插件可能需要更长时间。

**Q: 可以提交商业插件吗？**
A: 可以，但需要明确授权说明和使用条款。

**Q: 如何更新已发布的插件？**
A: 创建新的版本目录，更新配置文件，提交新的 PR。

**Q: 插件有 Bug 怎么办？**
A: 及时修复并更新版本，通过 GitHub Issues 跟踪问题。
